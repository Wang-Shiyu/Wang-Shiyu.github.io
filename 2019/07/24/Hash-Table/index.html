Overview

HashTable or HashMap (denoted as $$HashMap<K, V>$$) is a data structure that can map keys to values.

HashMap is unsynchronized and permits nulls(null values and null key), which means it is not thread safe.

HashMap makes no guarantees as to the order of the map, which means it does not return the keys and values in the same order they have been inserting into the HashMap.

Note:

  • Just always use substring instead of atChar to add chars to string.
  • to get the last char of a string, use string.substring(string.length() - 1));
  • Use the return value indexOf('a') to check if a single character appears in a string.
  • It’s not permitted to add the same element into a hash set.

Time Complexity:$ O(n^3) $: To verify the if characters with in index range [i,j) are all unique, we spend $$O(j-1)$$ time.

Space Complexity: $$O(min(n,m))$$ , m is the size of alphabet.

java Hash Set – Sliding Window Method

A sliding window is an abstract concept commonly used in array/string problems.

We use Hash set to store the characters in current window $$[i,j)$$, then slide the pointer j to the right until s[j] is already in the hash set. At this point, we record the maximum length of substring s[i-j] and remove s[i]from the left. If we do this for all i, we get the answer.

Time Complexity:$ O(2n) = O(n)$.

Space Complexity:$ O(min(m,n)) $: size of hash set is upper bounded by the size of string s and the size of alphabet.

But this solution is still not perfect. Removing s[i] 1 by 1 from the left is unnecessary. We can define a mapping of the characters to its index. In this wat, we can skip the repeated characters immediately by relocating i to j+1.

java Sliding Window Optimized

When confliction appears, move i to the character next to the duplicative character. Ignore the repeated situations that take place outside $$[i,j)$$

  • Time complexity : $$ O(n) $$
  • Space complexity : $$O(min(m, n))$$
